MOBILE APP DESIGN
I’m passionate about making everything look all the more eye-pleasing to make customers get a good feel of your brand.
I have also helped design brand identitities.
PROBLEM
Often times, we receive a follow/friend request from random accounts, not knowing with certainty if or how we know them or even if the account is genuine. This app tries to solve that problem by providing a measure of authenticity and also comparing profiles and displaying common things that stand out.
START SCREEN
The app starts off asking the user to search for a profile and describing both authenticity and relatedness.
SEARCH SCREEN
The search results are displayed as profile cards with name and username. It also has real-time search from the database as the user starts typing.
Relatedness Screen
The relatedness between the signed in profile and the one searched for is shown in this screen.

The purple theme for relatedness was caried out throughout this screen, including the card outline, authenticty button and background design. Even the relatedness result follows the pattern.

There is also a button that takes you to the authenticity screen fo the same user with the same gradient as the menu for consistency.
Authenticity Screen
The authenticity of the profile is computed using a unique algorithm that considers various parameters. It’s then displayed using a arched scale.

The red/pink theme for authenitcity is also caried out throughout the screen, including the card outline, relatedness button and background design. Even the authenticity result follows a pattern.

Similarily, there is also a button that takes you to the relatedness screen fo the same user.
Name / Logo
The name was chosen to be anonym-us as the app aims to find relatedness and authenticity information of anonymou users when one receives a folllow request
